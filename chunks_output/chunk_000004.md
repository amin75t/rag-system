# راه‌اندازی
## Cloud Langfuse
برای استفاده از Langfuse در محیط توسعه اگر مشکل محرمانگی اطلاعات وجود نداشته باشد می‌توان از Langfuse Cloud استفاده کرد.
https://cloud.langfuse.com
بعد از لاگین با ایجاد یک پروژه جدید، موارد زیر به دست می‌آید:
LANGFUSE_HOST
LANGFUSE_PUBLIC_KEY
LANGFUSE_SECRET_KEY
## Self-hosting Langfuse
بنا به دلایل مختلف از جمله محرمانگی اطلاعات، هزینه‌ی بالای SaaS  و نیاز به مقیاس‌پذیری، توصیه می‌شود که Langfuse به صورت خودمیزبان راه‌اندازی شود.
در سند رسمی زیر راه‌اندازی با استفاده از داکر شرح داده شده است:
https://langfuse.com/self-hosting/deployment/docker-compose
در صورت نیاز به مقیاس‌پذیری بالا لازم است هرکدام از سرویس‌های موردنیاز (از جمله ClickHouse و Redis و …) به صورت عناصر مجزا راه‌اندازی شوند.
